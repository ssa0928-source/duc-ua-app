**더블유카지노**

**신규유저 UA 크리에이티브 테스트 플랜**

Creative Testing Playbook — Full Cycle

| 작성 부서 | 마케팅팀 (UA/Performance) |
| :---- | :---- |
| **대상 앱** | 더블유카지노 (소셜카지노) |
| **테스트 대상** | 신규유저 (New User Acquisition) |
| **문서 목적** | 크리에이티브 테스트 전체 사이클 설계 및 인사이트 누적 체계 정의 |

# **1\. 테스트 철학 및 핵심 원칙**

## **1.1 이 플랜의 목표**

단순히 '좋은 소재'를 찾는 것이 아니라, 더블유카지노 신규유저에게 작동하는 크리에이티브 법칙(패턴)을 발굴하고 누적하는 것이다. 각 테스트 사이클이 끝날 때마다 위너 소재 1개가 아니라, 다음 가설을 더 정확하게 세울 수 있는 인사이트가 쌓여야 한다.

## **1.2 핵심 원칙**

* 변수 1개 원칙: 소재당 하나의 요소만 변경한다. 2개 이상 바꾸면 원인 파악 불가

* 고정값 원칙: 테스트하지 않는 요소는 모든 소재에서 픽셀 단위로 동일하게 유지

* 순차 진행 원칙: 훅 → 바디 → CTA → 조합 순서를 지킨다. 순서를 바꾸면 해석이 흐려짐

* 사전 판정 기준 원칙: 테스트 런칭 전에 승자 조건을 팀 내 합의로 확정한다

* 인사이트 누적 원칙: 위너가 나오면 반드시 '왜 이겼는가'를 기록하고 다음 가설에 반영한다

## **1.3 전체 테스트 구조**

훅, 바디, CTA를 순서대로 검증한 뒤 최적 조합을 완성한다.

| Phase | 목표 | 테스트 변수 | 고정 요소 |
| :---- | :---- | :---- | :---- |
| Phase 0 | 대조군(기준 소재) 확정 | 없음 (설계 단계) | 전체 |
| Phase 1 | 최적 훅 발굴 | 훅 유형 5종 | 바디 \+ CTA (Phase 0 고정값) |
| Phase 2 | 최적 바디 발굴 | 바디 유형 4종 | Phase 1 위너 훅 \+ CTA |
| Phase 3 | 최적 CTA 발굴 | CTA 문구/비주얼 3종 | Phase 1 위너 훅 \+ Phase 2 위너 바디 |
| Phase 4 | 최종 조합 검증 | 위너 조합 vs 대조군 | 전체 구조 확정 |
| Phase 5 | 스케일업 \+ 다음 사이클 | 비주얼 변형 / 신규 가설 | 위너 구조 유지 |

# **2\. Phase 0 — 고정값 소재(대조군) 정의**

## **2.1 목적**

모든 Phase에서 '변수가 아닌 것'을 고정하기 위한 기준 소재를 1개 제작한다. 이 소재는 전체 테스트 기간 동안 대조군으로 상시 운영되며, 최종 위너 소재의 성과 개선폭을 측정하는 기준점이 된다.

## **2.2 대조군 소재 구성 스펙**

| 구성 요소 | 내용 | 비고 |
| :---- | :---- | :---- |
| 훅 (0\~3초) | Big Win 모먼트 — 슬롯 터지는 순간 \+ 코인 카운트업 숫자 | 가장 보편적인 소셜카지노 훅 |
| 바디 (4\~18초) | 게임 플레이 영상 \+ 신규가입 XX코인 무료 텍스트 오버레이 | 보상 구체화 \+ 플레이 장면 조합 |
| CTA (19\~22초) | 지금 무료로 시작하기 버튼 강조 | 가장 범용적인 전환 문구 |
| 길이 | 20\~25초 |  |
| 이미지 포맷 | 1.91:1, 1:1, 4:5 |  |
| 영상 포맷 | 1:1, 9:16, 16:9 |  |
| 텍스트 위치 | 상단 1/3 내 고정 | 모든 테스트 소재 동일 |
| 폰트/컬러 | 브랜드 가이드 기준으로 확정 후 고정 | 확정 후 문서에 기재 |

| ⚠️  디자이너 주의사항 Phase 0 소재는 '잘 만드는 것'보다 '기준을 만드는 것'이 목적이다. 이 소재의 바디와 CTA 구성은 Phase 1\~3 전체에서 고정값으로 사용된다. 제작 완료 후 반드시 팀 리뷰를 거쳐 '고정값 확정' 승인을 받은 뒤 테스트를 시작한다. |
| :---- |

# **3\. Phase 1 — 훅(Hook) 테스트**

## **3.1 테스트 목적 및 가설**

목적: 소셜카지노 신규유저의 스크롤을 멈추는 감정 트리거가 무엇인지 파악한다.

| 핵심 가설 소셜카지노 신규유저는 '감정적 자극(대박 순간)'에 가장 먼저 반응하고, 그 다음으로 '즉각적 보상 제시'에 반응할 것이다. 반면 '궁금증 유발' 또는 'UGC 스타일'은 게임 카테고리 특성상 상대적으로 낮은 반응을 보일 것이다. |
| :---- |

## **3.2 소재 구성 원칙**

* 고정 요소: Phase 0에서 확정한 바디(4\~18초) \+ CTA(19\~22초) 동일 사용

* 변수 요소: 훅(0\~3초) 유형만 교체

* 총 소재 수: 훅 유형 5개 \= 소재 5개

* 모든 소재의 바디/CTA는 픽셀 단위로 동일하게 유지 (편집본 공유 필수)

## **3.3 테스트 훅 유형 5종**

| \# | 훅 유형 | 비주얼 설명 | 감정 트리거 |
| :---- | :---- | :---- | :---- |
| 1 | Big Win 모먼트 | 슬롯 터지는 순간 \+ 코인 폭발 \+ 숫자 카운트업 | 흥분 / 탐욕 |
| 2 | 무료 보상 직접 제시 | '지금 가입하면 OO코인 무료' 첫 화면 텍스트 \+ 코인 비주얼 | 이득 / 손실 회피 |
| 3 | 궁금증 유발 (클리프행어) | 당첨 결과만 먼저 보여주고 컷 — '이거 진짜야?' 자막 | 호기심 / FOMO |
| 4 | 실제 플레이 POV | 손가락 스핀 → 당첨 직전에서 컷 (1인칭 시점) | 몰입감 / 대리경험 |
| 5 | UGC 리액션 | 당첨 순간 놀라는 표정 (실사 or 연출), 자연스러운 반응 | 공감 / 신뢰 |

## **3.4 위너 선정 기준**

| 구분 | 지표 | 기준값 | 비고 |
| :---- | :---- | :---- | :---- |
| 1차 판정 (필수) | CTR (Click-through Rate) | 대조군 \+20% 이상 | 주 판정 지표 |
| 2차 판정 (보조) | IPM (Installs per Mille) | 대조군 \+15% 이상 | CTR 동점 시 활용 |
| 판정 시점 | 노출량 | 소재당 10,000 이상 도달 후 | 미달 시 판정 보류 |
| 운영 기간 | 최소 런 기간 | 7일 이상 | 초반 알고리즘 노이즈 제거 |
| 이월 기준 | 다음 Phase 진입 | 상위 2개 훅 유형 | 예산 부족 시 1개로 좁혀도 무방 |

| Phase 1에서 IPM이 낮은데 CTR이 높은 소재가 나올 경우: → '클릭은 하는데 설치 안 함' \= 훅은 흥미롭지만 바디/CTA가 기대치를 충족 못 하는 신호 → 이 패턴은 Phase 2에서 바디 가설로 전환하여 검증한다 |
| :---- |

## **3.5 플랫폼별 운영 방식**

* Meta: Creative Testing 캠페인 또는 동일 Ad Set 내 소재 5개 동시 런칭

* UAC / AppLovin: 각각 별도 캠페인으로 분리 운영 (플랫폼 알고리즘 차이로 크로스 비교 금지)

* 플랫폼별로 위너가 다를 수 있음 — 각각 기록 후 공통 패턴 추출

# **4\. Phase 2 — 바디(Body) 테스트**

## **4.1 테스트 목적 및 가설**

목적: 훅 이후 어떤 내용 구성이 신규유저를 설치까지 설득하는지 파악한다.

| 핵심 가설 소셜카지노 신규유저는 '게임의 다양성'보다 '보상의 구체성'을 보여주는 바디에서 더 높은 설치 전환율을 보일 것이다. 단, '긴박감(한정 혜택)'이 보상 구체화보다 더 강하게 작동할 가능성도 검증이 필요하다. |
| :---- |

## **4.2 소재 구성 원칙**

* 고정 요소: Phase 1 위너 훅 (1개로 좁혀서 사용 권장)

* 위너 훅 2개를 모두 사용하면 소재 수가 2배 → 예산/리소스 확인 필요

* 고정 요소: Phase 0에서 확정한 CTA 동일 사용

* 변수 요소: 바디(4\~18초) 유형만 교체

* 총 소재 수: 바디 유형 4개 \= 소재 4개 (위너 훅 1개 기준)

## **4.3 테스트 바디 유형 4종**

| \# | 바디 유형 | 내용 구성 | 검증 포인트 |
| :---- | :---- | :---- | :---- |
| 1 | 보상 구체화 | 코인 수량 \+ 보너스 혜택 \+ 첫 충전 혜택 텍스트 나열 | 숫자의 구체성이 전환을 만드는가 |
| 2 | 슬롯 다양성 | 슬롯 종류 빠르게 훑기 \+ 'N가지 슬롯' 텍스트 강조 | 선택지의 풍부함이 설득하는가 |
| 3 | 긴박감 (한정성) | 카운트다운 타이머 \+ '오늘만' \+ 한정 배너 비주얼 | 희소성이 즉시 설치를 유도하는가 |
| 4 | 소셜 증거 | '매일 OO명 플레이 중' \+ 유저 반응 텍스트 모음 | 다수의 행동이 안심을 주는가 |

## **4.4 위너 선정 기준**

| 구분 | 지표 | 기준값 | 비고 |
| :---- | :---- | :---- | :---- |
| 1차 판정 (필수) | CPI (Cost per Install) | 대조군 \-15% 이하 | 주 판정 지표 |
| 2차 판정 (필수) | D3 Retention | 대조군 \-10%p 이상 이탈 없을 것 | 반드시 병행 확인 |
| 보조 확인 | IPM 유지 여부 | Phase 1 위너 IPM 대비 \-10% 이내 | 훅 효과 유지 확인 |
| 판정 시점 | 노출량 | 소재당 10,000 이상 도달 후 |  |
| 운영 기간 | 최소 런 기간 | 7일 이상 \+ D3 데이터 수집까지 대기 |  |

| D3 Retention을 반드시 함께 보는 이유: → 바디에서 게임 기대치를 과장하거나 잘못 심으면 설치 후 3일 내 이탈이 급증한다. → CPI가 낮아도 Retention이 무너지면 LTV 관점에서 최악의 소재다. → CPI 낮음 \+ D3 Retention 정상 \= 진짜 위너 / CPI 낮음 \+ D3 Retention 급락 \= 폐기 |
| :---- |

# **5\. Phase 3 — CTA 테스트**

## **5.1 테스트 목적 및 가설**

목적: 전환을 최종 결정짓는 마지막 문구와 비주얼이 무엇인지 파악한다.

| 핵심 가설 '무료'를 강조하는 CTA가 '즉시성' 또는 '한정성'을 강조하는 CTA보다 소셜카지노 신규유저의 설치 전환율에서 우위를 보일 것이다. 단, 소재 피로도가 누적된 유저에게는 '한정 혜택' CTA가 더 강하게 작동할 수 있다. |
| :---- |

## **5.2 소재 구성 원칙**

* 고정 요소: Phase 1 위너 훅 \+ Phase 2 위너 바디

* 변수 요소: CTA 문구 \+ 버튼 비주얼 (19\~22초 구간만 교체)

* 총 소재 수: CTA 유형 3개 \= 소재 3개

## **5.3 테스트 CTA 유형 3종**

| \# | CTA 유형 | 문구 예시 | 심리 기제 |
| :---- | :---- | :---- | :---- |
| 1 | 무료 강조 | 지금 무료로 받기 | 이득 심리 / 진입장벽 제거 |
| 2 | 즉시성 강조 | 지금 바로 플레이 | 즉각 행동 유도 / FOMO |
| 3 | 한정 혜택 강조 | 오늘만 신규 보너스 지급 | 희소성 / 손실 회피 |

## **5.4 위너 선정 기준**

| 구분 | 지표 | 기준값 | 비고 |
| :---- | :---- | :---- | :---- |
| 1차 판정 (필수) | CVR (Click → Install 전환율) | 평균 대비 \+10% 이상 | 주 판정 지표 |
| 2차 판정 (보조) | CPI | 평균 대비 \-10% 이하 | 보조 지표 |
| 판정 시점 | 클릭 수 | 소재당 클릭 500 이상 | CVR 판정 최소 샘플 |
| 운영 기간 | 최소 런 기간 | 7일 이상 |  |

# **6\. Phase 4 — 최종 조합 검증**

## **6.1 목적**

Phase 1\~3의 각 단계 위너를 조합한 소재가 Phase 0 대조군 대비 유의미하게 우월한지 최종 검증한다. 각 단계에서 독립적으로 이긴 요소들이 결합했을 때도 시너지를 내는지 확인하는 과정이다.

## **6.2 소재 구성**

| 소재 | 구성 | 역할 |
| :---- | :---- | :---- |
| 위너 조합 소재 A | \[Phase 1 위너 훅\] \+ \[Phase 2 위너 바디\] \+ \[Phase 3 위너 CTA\] | 최종 위너 후보 |
| 위너 조합 소재 B\~C | 위너 구조 유지 \+ 비주얼만 교체한 변형 2개 | 소재 피로도 대비 |
| 대조군 (Phase 0\) | 테스트 시작 시점부터 상시 운영 중인 기준 소재 | 성과 비교 기준 |

## **6.3 최종 위너 판정 기준**

| 지표 | 목표 | 비고 |
| :---- | :---- | :---- |
| CPI | 대조군 대비 \-15% 이상 개선 | 핵심 판정 지표 |
| D7 ROAS | 대조군 대비 동등 이상 | 품질 검증 |
| D3 Retention | 대조군 대비 \-5%p 이내 | LTV 품질 유지 확인 |
| IPM | 대조군 대비 \+20% 이상 유지 | 볼륨 확장 가능성 확인 |

| 조합 소재가 대조군보다 유의미하게 개선되지 않은 경우: → 단계별 위너가 조합 시 상충하는 구조일 가능성 있음 → Phase 인사이트 로그를 재검토하여 가설 충돌 여부 확인 → 훅 또는 바디를 2위 후보로 교체한 조합 소재를 추가 테스트 고려 |
| :---- |

# **7\. Phase 5 — 스케일업 및 다음 사이클**

## **7.1 스케일업 액션**

* 예산 집중: 위너 조합 소재 \+ 비주얼 변형본(B, C)으로 볼륨 확대

* 플랫폼 확장: Meta 위너 → UAC / AppLovin / TikTok에 크로스 테스트

* 소재 수명 모니터링: 위너 소재 IPM이 초기 대비 30% 이상 하락 시 '소재 피로' 신호로 판단

## **7.2 다음 사이클 가설 시드 발굴**

Phase 4까지 완료 후 다음 사이클은 '새로운 탐색'이 아니라 '검증된 법칙의 확장'이어야 한다. 인사이트 로그에서 미검증 가설을 추출하여 Phase 1부터 재진입한다.

| 재진입 Phase | 새로운 테스트 방향 예시 |
| :---- | :---- |
| Phase 1 재진입 | 시즌/이벤트 한정 훅 테스트 (명절, 특별 이벤트 연계) |
| Phase 1 재진입 | 신규 슬롯 출시 연계 훅 (신작 슬롯 첫 공개) |
| Phase 2 재진입 | 바디 내 텍스트 밀도 테스트 (텍스트 많음 vs 비주얼 중심) |
| Phase 2 재진입 | 바디 길이 테스트 (10초 vs 15초) |
| 신규 Phase 추가 | 포맷 테스트 (9:16 vs 1:1 vs 16:9) |
| 신규 Phase 추가 | 플랫폼 전용 소재 (TikTok 네이티브 스타일) |

# **8\. 인사이트 누적 체계**

## **8.1 왜 인사이트 누적이 필요한가**

테스트가 단순 반복이 되지 않으려면, 위너 소재가 아니라 위너 패턴(법칙)을 추출해야 한다. 각 Phase 종료 시 반드시 '왜 이겼는가'를 팀 내에서 합의하고 문서화한다. 이것이 6사이클 쌓이면 더블유카지노만의 크리에이티브 플레이북이 된다.

| 단순 반복:  훅A가 이겼다 → 다음엔 훅F, G, H 테스트 (매번 처음부터 탐색) 인사이트 누적:  훅A가 이긴 이유 \= '숫자 카운트업 \+ 당첨 직전 컷' 패턴                 → 다음 가설: 이 패턴을 더 과장하면? 다른 게임에 적용하면?                 → 그 다음 가설: 반박 — 혹시 카운트업이 아니라 효과음 때문이었나? |
| :---- |

## **8.3 크리에이티브 플레이북 누적 구조**

인사이트 로그가 쌓이면 아래 구조로 플레이북을 업데이트한다.

| 플레이북 항목 | 내용 예시 (누적될 내용) |
| :---- | :---- |
| 작동하는 훅 패턴 | 숫자 카운트업 \+ 당첨 직전 컷 / 첫 화면 코인 폭발 |
| 작동하지 않는 훅 패턴 | 텍스트 과다 훅 / 배경음 없는 무음 훅 |
| 작동하는 바디 구조 | 보상 수량 구체화 \+ 플레이 장면 조합 |
| 유저 심리 인사이트 | 소셜카지노 유저는 '다양성'보다 '내가 받을 것'에 반응 |
| 플랫폼별 차이 | Meta: UGC 반응 좋음 / UAC: 게임 플레이 직접 노출 강함 |
| 소재 수명 패턴 | 위너 소재 평균 수명 OO주, 이후 IPM OO% 하락 |

# **9\. 디자이너 공통 가이드**

## **9.1 제작 원칙**

* 변수는 소재당 1개만: 2개 이상 바꾸면 어느 것이 성과를 만들었는지 알 수 없음

* 고정값은 픽셀 단위로 동일하게 유지: Phase 0 소재의 바디/CTA를 그대로 복붙

* Phase 0\~3은 '잘 만들기'가 아니라 '가설 검증'이 목적: 퀄리티 업그레이드는 Phase 4 위너 확정 후

* 텍스트 가독성 확보: 무음 재생 환경 고려, 자막 또는 텍스트 오버레이 필수
