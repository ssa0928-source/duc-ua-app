# 공개 접속 & URL 고정 가이드

## 핵심 요약

| 목표 | 방법 |
|------|------|
| **누구나 접속** | HTTPS로 배포(Netlify, Vercel 등) + 앱이 쓰는 Supabase는 **anon 키**로 접근(이미 프론트에 포함). DB는 **RLS**로 허용 범위만 열어두기. |
| **업데이트해도 링크 유지** | **항상 같은 호스팅 “사이트(프로젝트)”**에만 배포. 새 사이트를 만들면 URL이 달라짐. |
| **브랜드 주소** | **커스텀 도메인** 1회 연결 → 이후 배포는 같은 도메인으로만 갱신됨. |

---

## 1. 링크가 바뀌는 대표적인 실수

- Netlify에서 **매번 “새 사이트 추가”** 후 dist만 드래그 업로드 → 매번 `xxxx.netlify.app`가 달라질 수 있음.
- **해결:** [Netlify](https://app.netlify.com)에서 **기존 사이트 하나**를 골라 **Deploys**에서만 다시 배포(또는 Git 연동 시 push).

---

## 2. 권장: Git 연동 + 자동 배포 (URL 고정)

1. 코드를 **GitHub / GitLab / Bitbucket**에 저장소로 올림.
2. Netlify(또는 Vercel)에서 **Add new site → Import an existing project**로 그 저장소 연결.
3. 빌드 설정 예시:
   - **Build command:** `npm run build`
   - **Publish directory:** `dist`
4. 이후 **`git push`할 때마다 같은 사이트 URL**로 자동 배포됨.

이 저장소에는 `netlify.toml`이 있어 Netlify가 위 설정을 자동 인식할 수 있습니다.

---

## 3. 커스텀 도메인 (가장 안정적인 주소)

- 회사 도메인 예: `ua.doubleugames.com`
- Netlify/Vercel **Domain settings**에서 도메인 추가 후 DNS(CNAME/A)만 안내에 맞게 설정.
- **한 번 연결하면** 이후 배포는 전부 **같은 도메인**으로 반영됩니다.

---

## 4. 환경 변수 (팀·공개 배포)

- `.env`는 Git에 올리지 마세요 (`.gitignore` 유지).
- Netlify/Vercel **Site settings → Environment variables**에 예:
  - `VITE_SUPABASE_URL`
  - `VITE_SUPABASE_ANON_KEY`
  - `VITE_GEMINI_API_KEY` (사용 시)
- 배포 후 **Rebuild** 해야 빌드에 반영되는 경우가 많습니다.

---

## 5. Supabase “누구나 읽기/쓰기” 주의

- 프론트의 anon 키는 브라우저에 노출됩니다. **RLS(Row Level Security)**로 테이블별로 허용 정책을 두는 것이 안전합니다.
- “내부용이지만 링크만 공유” 수준이면, 익명 `SELECT/INSERT` 허용 범위를 PRD에 맞게 최소화하세요.

---

## 6. Vercel을 쓰는 경우

- [vercel.com](https://vercel.com)에서 **하나의 프로젝트**에 Git 연동.
- Framework Preset: **Vite**, Build: `npm run build`, Output: `dist`.
- 프로덕션 URL은 `프로젝트명.vercel.app`으로 고정, 커스텀 도메인 동일하게 적용 가능.

---

**정리:** “같은 호스팅 사이트 + (가능하면) Git 자동 배포 + (선택) 커스텀 도메인”이면 업데이트해도 **공유 링크는 그대로** 유지할 수 있습니다.
