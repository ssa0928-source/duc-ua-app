# 03. 자동화 시스템 설계

---

## 시스템 목표

가설 생성 → 크리에이티브 제작 → 성과 분석으로 이어지는 흐름을 자동으로 연결

---

## 기술 스택

| 구성 요소 | 선택 | 비고 |
|-----------|------|------|
| DB | Supabase (무료 티어) | 가설 데이터 저장 |
| 프론트엔드 | React 웹앱 | Cursor로 개발 중 |
| 개발 도구 | Cursor | 미완성 상태 |
| AI 연동 | Claude API | 웹앱 안정화 이후 도입 예정 |

---

## 핵심 블로커: 네이밍 컨벤션

### 문제
- 기존 소재 네이밍에 가설 ID가 포함되어 있지 않음
- 가설 ID 없이는 성과 데이터와 가설을 자동 매칭 불가

### 해결안
소재명 뒤에 가설 ID 태그 추가:

```
기존: [소재명]
변경: [소재명]_H01_TEST  (테스트군)
      [소재명]_H01_CTRL  (대조군)
```

### 실행 조건
- 마케팅팀과 협의 필요 (네이밍 룰 변경은 마케팅팀 캠페인 운용에 영향)
- 공식 구조 정립 이후 진행 권장

---

## 자동화 구현 로드맵

### 단계 1 (현재): 웹앱 코어 개발
- Supabase + React 기본 구조
- 가설 입력/조회/관리 기능
- Cursor로 개발 중 (미완성)

### 단계 2 (웹앱 안정화 후): AI 기능 연동
- Claude API 키 발급
- 가설 자동 생성/분류 지원

### 단계 3 (문서 포맷 표준화 후): Confluence 연동
- Confluence API로 기획 문서에서 가설 자동 추출
- 문서 포맷 표준화가 선행되어야 함

---

## 설계 원칙

> **자동화 순서가 중요**: Confluence API 연동과 AI 기능은 웹앱이 안정화되고 문서 포맷이 표준화된 이후에 도입. 순서를 역전시키면 유지보수 비용이 급증.

---

## 현재 상태 (2026년 3월 기준)

- 웹앱: Cursor 개발 중, 미완성
- 네이밍 컨벤션 변경: 마케팅팀 협의 전
- Claude API: 도입 미정 (웹앱 안정화 후)
- Confluence API: 도입 미정 (문서 포맷 표준화 후)
